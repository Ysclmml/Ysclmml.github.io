Go反射总结

Go 语言中反射的操作主要定义在标准库 reflect 中，在标准库中定义了两种类型来表现运行时的对象信息，分别是：reflect.Value（反射对象的值信息）和 reflect.Type（反射对象的类型信息），Go 语言中所有反射操作都是基于这两个类型进行的。

0. 准备

下面所有反射操作都是基于下面结构体进行的.

定义了一个User结构体, 并且包含一个方法. 

type User struct{
	Id int64
	Name string `form:"name" json:"name" binding:"required,gte=5" message:"required:用户名不能为空"`
	Age uint8 `form:"age" json:"age" binding:"required,lt=10" message:"required:年龄不能为空|lt:年龄必须小于10"`
}

func (u User) GetName() string {
	return "User: " + u.Name
}

1. Value与Type

前面说过reflect包提供了2个对象来描述go语言运行时的对象状态信息. 那么首先就需要通过reflect包提供的两个基础方法reflect.ValueOf() 和 reflect.TypeOf()来获取到这两个对象.

基础使用

u := User{1, "tom", 20}
v := reflect.ValueOf(u)
fmt.Println(v)  // 输出: {1 tom 20}
t := reflect.TypeOf(u)
fmt.Println(t)  // 输出: main.User

根据基础的打印信息, 可以看出已经知道了u对象的值及其类型信息. 接下来就是需要指导这个对象内更具体的信息. 

fmt.Println(v.Type(), t, v.Type() == t) // main.User main.User true

type和value可以互相转换

fmt.Println(v.Type(), t, v.Type() == t) // main.User main.User true
fmt.Println(reflect.New(t))  // &{0  0}

通过v.Type()方法可以获取到value对象, 通过relect包的New方法可以获取到Type对象所对应的零值对象.

2. Kind

Kind 表示反射对象的类型 Type 所代表的具体类型，零值表示无效的类型，下面是reflect包预定义的Kind

const (
	Invalid Kind = iota
	Bool
	Int
	Int8
	Int16
	Int32
	Int64
	Uint
	Uint8
	Uint16
	Uint32
	Uint64
	Uintptr
	Float32
	Float64
	Complex64
	Complex128
	Array
	Chan
	Func
	Interface
	Map
	Ptr
	Slice
	String
	Struct
	UnsafePointer
)

上面的Value和Type对象都有Kind()方法来获取对应的真实类型

fmt.Println(v.Kind())  // struct
fmt.Println(t.Kind())  // struct

通过Type对象和Value对象获取到的Kind可以看到都是struct类型

3. 反射对象的字段

首先需要说明, 这里所说的对象一般都是指代结构体(struct)对象.

使用反射一般来说是需要获取到反射的对象的字段信息.

NumField

Value对象和Type对象都有NumField这个方法来获取到结构体对象的字段数量

使用这个方法必须要求被反射的原始对象必须为结构体, 即使是一个指向结构体对象的指针也会panic错误. 

u := User{1, "tom", 20}
fmt.Println(reflect.ValueOf(u).NumField())   // 3

Field

获取到了数量之后, 可以通过for循环来遍历所有字段, 使用Field(i)方法来获取到对应的字段信息.

Type对象获取到的对应的字段是结构体对象定义的元信息, 如tag信息, 字段名字, 字段类型等…

Value对象获取到的是对应字段的值

u := User{1, "tom", 20}
	// fmt.Println(reflect.ValueOf(u).NumField())
	t := reflect.TypeOf(u)
	v := reflect.ValueOf(u)
	for i := 0; i < t.NumField(); i++ {
		typeField := t.Field(i)
		valueField := v.Field(i)
		fmt.Println("typeField", typeField)
		fmt.Println("valueField", valueField)
	}

最后输出如下

typeField {Id  int64  0 [0] false}
valueField 1
typeField {Name  string form:"name" json:"name" binding:"required,gte=5" message:"required:用户名不能为空" 8 [1] false}
valueField tom
typeField {Age  uint8 form:"age" json:"age" binding:"required,lt=10" message:"required:年龄不能为空|lt:年龄必须小于10" 24 [2] false}
valueField 20

可以看到typeField输出了结构体定义时的完整的信息

这个typefield是StructField类型, 在reflect包中定义如下

type StructField struct {
	// Name is the field name.
	Name string
	// PkgPath is the package path that qualifies a lower case (unexported)
	// field name. It is empty for upper case (exported) field names.
	// See https://golang.org/ref/spec#Uniqueness_of_identifiers
	PkgPath string

	Type      Type      // field type
	Tag       StructTag // field tag string
	Offset    uintptr   // offset within struct, in bytes
	Index     []int     // index sequence for Type.FieldByIndex
	Anonymous bool      // is an embedded field
}

主要关注Tag, Name, Type参数分别是字段的tag, 名字及其类型, 利用这几个参数就尤其是可以通过tag来实现很多自动化的工作了. 如gin框架中的校验器使用的validate tag以及序列化所使用到的json tag都是利用反射获取tag信息来完成的.

上面的valueField返回的还是一个Value对象, 通过这个Value对象可以获取每个字段的值, 如果字段仍然是struct类型, 那么可以继续递归操作.

4. Elem

Elem() 方法定义：func (v Value) Elem() Value，返回 v 包含的值或指针 v 指向的值，v 的 Kind 如果不是 Interface 或 Ptr，则会 panic. Elem方法等效于对指针类型变量做了一个*操作, 不过这个也有不同的地方.

当需要对反射的对象做修改操作时, 就必须用到Elem方法. 举一个demo. 

u := User{1, "tom", 20}
	fmt.Println("before u: ....", u)
	elem := reflect.ValueOf(&u).Elem()
	for i := 0; i < elem.NumField(); i++ {
		eleField := elem.Field(i)
		if eleField.CanSet() {
			if eleField.Kind() == reflect.String {
				eleField.SetString("修改后String")
			} else if eleField.Kind() == reflect.Int {
				eleField.SetInt(333333)
			} else if eleField.Kind() == reflect.Int64 {
				eleField.SetInt(6666666)
			}
		}
	}
	fmt.Println("after u: ....", u)

输出结果

before u: .... {1 tom 20}
after u: .... {6666666 修改后String 20}

上面使用到了CanSet方法, 这个是只有使用elem方法获取到field才会返回True, 才可以做后面的修改值操作.

总而言之, elem方法其实就是相当于对获取到的字段做了一次取指针操作, 与直接使用没有太大的区别.

5. 反射对象的方法

获取方法对象和获取属性很类似, 使用方法都在下面的注释说明了. 不过里面需要注意的是方法入参还把接收对象也当做入参参数之一了. 

u := User{1, "tom", 20}
t := reflect.TypeOf(u)
v := reflect.ValueOf(u)
for i := 0; i < v.NumMethod(); i++ {
	method := t.Method(i)  // 返回方法结构体
	vMethod := v.Method(i)
	mt := method.Type
	fmt.Println(method, vMethod, "=====", method.Type)

	// 获取入参参数, 结构体对象的方法, 获取的方法信息对象会把方法的接受者也当着入参之一
	for j := 0; j < mt.NumIn(); j++ {
		in := mt.In(j)
		fmt.Println("in .......", in, in.Name(), in.Kind())  // in ....... main.User User struct
	}

	// 获取方法返回类型
	for j := 0; j < mt.NumOut(); j++ {
		fmt.Println("method out type:", mt.Out(j))
	}

	// 方法的调用
	if method.Name == "SetName" {
		// 这里需要组织入参参数, 都是value类型的, 需要注意要传接收对象
		ins := []reflect.Value{reflect.ValueOf(u), reflect.ValueOf("反射调用方法值")}
		method.Func.Call(ins)
	}
}

6. 其他方法

FieldByName 直接通过名字获取到对应的字段信息

MethodByName 直接通过方法名字获取到对应的方法结构体

上面两种方式获取到结构体对象与遍历Field(index)的效果是一样的. 具体使用哪种看需求.

Interface 是value对象的方法. 通过这个interface可还原取到value对象的原始值

u := User{1, "tom", 20}
t := reflect.TypeOf(u)
v := reflect.ValueOf(u)
if name, exist := t.FieldByName("Name"); exist {
	fmt.Println(name.Name, name.Tag)
}
if fun, exist := t.MethodByName("SetName"); exist {
	// 调用方法
	ins := []reflect.Value{reflect.ValueOf(u), reflect.ValueOf("反射调用方法值")}
	fun.Func.Call(ins)
}
// 使用Interface方法还原对象
if u1, ok := v.Interface().(User); ok {
	fmt.Println("after:", u1.Name, u1.Age)
}
