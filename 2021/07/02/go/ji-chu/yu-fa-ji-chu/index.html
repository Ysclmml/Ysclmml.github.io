数据类型

上面是go的基础类型, 没啥好多说的, 不过还是有几个需要注意的地方.

  • int类型和操作系统有关, 32位机器上是4个字节长度的有符号整型, 64位机器是8个字节的有符号整型. int和unit与 int8,int16不是一个类型.
  • byte是unit8的别名, rune是int32的别名, 这两者是等价的.
  • struct和array都是值类型, slice才是引用类型

iota

iota是go语言的常量计数器，只能在常量的表达式中使用。 iota在const关键字出现时将被重置为0。const中每新增一行常量声明将使iota计数一次(iota可理解为const语句块中的行索引)。 使用iota能简化定义，在定义枚举时很有用。

// 测试常量
// const中每新增一行常量声明将使iota计数一次(iota可理解为const语句块中的行索引)。
const (
	_  = iota
	KB = 1 << (10 * iota)
	MB = 1 << (10 * iota)
	GB = 1 << (10 * iota)
	TB = 1 << (10 * iota)
	PB = 1 << (10 * iota)
)
fmt.Println(KB, PB)

类型转换

// 浮点型与整型的互转
var x = 6.4
fmt.Println(int(x))

// []byte/rune类型和字符串类型的转换
d := "中国"
e := []byte(d)
f := []byte(d)
fmt.Println(d, e, f)

// 数组转切片
var a = [8]int{1,2,3,4}
var b []int
b = a[:]
fmt.Println(b)

// string转int Atoi 等价于 ParseInt(s, 10, 0)
i, _ := strconv.Atoi("12345")
fmt.Println(i)

// string转int64, 第二个参数是进制, 第三个参数是返回的位数, 0为int 8, 32, 64对应int8, int32...
// 报名记法就是str + convert的缩写
i2, _ := strconv.ParseInt("7", 10, 0)
fmt.Println("i2", i2)

// int64转字符串
s := strconv.FormatInt(65, 10)
fmt.Println(s)

// int转字符串, 相当于上面的一个简写
s = strconv.Itoa(12)
fmt.Println(s)

// float转字符串
s = strconv.FormatFloat(1.2456, 'E', -5, 64) // -> 1.2456E+00
fmt.Println(s)
s = strconv.FormatFloat(1.2456, 'f', -5, 64) // -> 1.2456
fmt.Println(s)

// 单个字节转字符串
chr := 'a'
fmt.Println(string([]rune{chr}))
// 转换的另一种方式
var cc byte = 'a'
s = fmt.Sprintf("%c", cc)
fmt.Println(s)

字符串修改

go里的字符串是可变的, 需要先转换成字节数组, 再进行转换

s := "我爱我家"
b := []rune(s)
b[3] = '变'
s = string(b)
fmt.Println(s)

数组

  1. 数组：是同一种数据类型的固定长度的序列。
  2. 数组定义：var a [len]int，比如：var a [5]int，数组长度必须是常量，且是类型的组成部分。一旦定义，长度不能变。
  3. 长度是数组类型的一部分，因此，var a[5] int和var a[10]int是不同的类型。
  4. 数组可以通过下标进行访问，下标是从0开始，最后一个元素下标是：len-1
    for i := 0; i < len(a); i++ {}
    for index, v := range a {}
  5. 访问越界，如果下标在数组合法范围之外，则触发访问越界，会panic
  6. 数组是值类型，赋值和传参会复制整个数组，而不是指针。因此改变副本的值，不会改变本身的值。
    7.支持 “==”、”!=” 操作符，因为内存总是被初始化过的。
    8.指针数组 [n]*T，数组指针 *[n]T
var a [5]int

// 遍历
for i := 0; i < len(a); i++ {
	fmt.Print(a[i], " ")
}
for i, val := range a {
	fmt.Println(i, val)
}

// 定义初始化
b := [3]string{"aaa", "bbb"}
fmt.Println(b)

// 自动根据后面初始化的个数来确定长度, 使用:指定索引初始化
data := [...]int{0, 1, 2, 3, 4, 10: 0}
fmt.Println(data)  // -> [0 1 2 3 4 0 0 0 0 0 0]

// 二维数组的定义与初始化
c := [3][3]int{{1, 2, 3}, {4, 5, 6}, {7, 8, 9}}
fmt.Println(c)
for _, row := range c {
	for _, val := range row {
		fmt.Print(val, " ")
	}
	fmt.Println()
}

// 数组可以使用比较运算符 == != 
d := [3]int{1, 2, 3}
e := [3]int{1, 2, 3}
f := [3]int{1, 3, 2}
fmt.Println(d == e, e== f)  // -> true false

切片

  1. 切片：切片是数组的一个引用，因此切片是引用类型。但自身是结构体，值拷贝传递。
  2. 切片的长度可以改变，因此，切片是一个可变的数组。
  3. 切片遍历方式和数组一样，可以用len()求长度。表示可用元素数量，读写操作不能超过该限制。
  4. cap可以求出slice最大扩张容量，不能超出数组限制。0 <= len(slice) <= len(array)，其中array是slice引用的数组。
  5. 切片的定义：var 变量名 []类型，比如 var str []string var arr []int。
  6. 如果 slice == nil，那么 len、cap 结果都等于 0。

定义与声明

通过make创建时, 需要指定切片的长度, 也可以指定第三个参数, 切片底层数组的长度, 也是切片的cap数量. 

// 切片声明
	var s1 []int
	fmt.Println(s1 == nil)

	// 不指定长度就是切片, 定义并初始化
	s2 := []int{1, 2, 3}
	fmt.Println(s2)

	// make创建
	s3 := make([]int, 3, 10)
	s4 := make([]int, 5)
	fmt.Println(s3, s4)

从数组中获取

go的这种取法和Python列表的切片方式一模一样, 左开右闭. 

// 数组中获取
arr := [8]int{1,2,3,4,5,6,7,8}
// 省略左边表示从0开始取, 省略右边表示取到结尾
s5 := arr[1:4]
s6 := arr[:]
s7 := arr[:5]
s8 := arr[1:]
fmt.Println(s6, s7, s8)

// 当然也可以直接从切片本身 切
fmt.Println(s5[0:2])

其他操作

// 切片的其他操作
// 切片是引用类型
arr := [8]int{1,2,3,4,5,6,7,8}
s1 := arr[:]
fmt.Println(s1)
arr[0] = 11
fmt.Println(s1)

// 扩充切片
s2 := []int{0, 0}
s2 = append(s2, 1, 2, 3)
fmt.Println(s2)

// 切片的复制, 可以指定将源切片复制到目标切片的指定位置.
s4 := []int{1, 2, 3, 4, 5, 6}
s3 := []int{-1, -1, -1, -1}
n := copy(s4[3:5], s3)
fmt.Println(n)
fmt.Println(s4)

// 切片的索引操作是可以取超出本身长度的, 但是不能超过其capacity
var a = []int{1, 3, 4, 5}
fmt.Printf("slice a : %v , len(a) : %v\n", a, len(a))
b := a[1:2]
fmt.Printf("slice b : %v , len(b) : %v, cap(c) : %v\n", b, len(b), cap(b))
c := b[0:3]
fmt.Printf("slice c : %v , len(c) : %v, cap(c) : %v\n", c, len(c), cap(b))

d := make([]int, 5, 10)
fmt.Println(len(d), cap(d))
copy(d, []int{1,2,3,4,5})
fmt.Println(d)
fmt.Println(d[2:6])

切片有len和cap两个属性, 切片本身重新resize的时候是可以超出自己len边界的,但是不可以超出cap

Map

map就是字典类型的数据结构, 比较简单, 不多做说明

// map定义可以指定数量, 也可以不指定具体数量
m := make(map[string]string)

// 赋值, 和Python里面一样
m["a"] = "aaa"
m["b"] = "bbb"

// 判断是否存在某key
if val, ok := m["a"]; ok {
	fmt.Println(val)
}

// 遍历
for key, val := range m {
	fmt.Println(key, val)
}

// 删除键
delete(m, "a")
delete(m, "aaaaaaaaa")
fmt.Println("==============")
for key, val := range m {
	fmt.Println(key, val)
}

一个和map相关的面试题

type student struct {
	name string
	age  int
}

m2 := make(map[string]*student)
stus := []student{
	{name: "pprof.cn", age: 18},
	{name: "测试", age: 23},
	{name: "博客", age: 28},
}

for _, stu := range stus {
	m2[stu.name] = &stu
}
for k, v := range m2 {
	fmt.Println(k, "=>", v.name)
}

上面结果比较出乎意料, 打印出来的name都是同一个.

原因其实也比较好理解, 因为结构体在go里面是值类型. for range循环遍历结构体切片的时候每次将结构体copy临时变量, 最后map中保存的其实都是for循环的那个变量的地址, 而不是切片内包含的结构体地址.

结构体

Go语言中没有“类”的概念，也不支持“类”的继承等面向对象的概念。Go语言中通过结构体的内嵌再配合接口比面向对象具有更高的扩展性和灵活性。

结构体可以模拟出面向对象编程的封装的特性(将属性和方法封装起来).

结构体定义

结构体定义的格式比较简单.

type 类型名 struct {
    字段名 字段类型
    字段名 字段类型
    …
}
  1. 类型名：标识自定义结构体的名称，在同一个包内不能重复。
  2. 字段名：表示结构体字段名。结构体中的字段名必须唯一。
  3. 字段类型：表示结构体字段的具体类型。

方法定义

Go语言中的方法（Method）是一种作用于特定类型变量的函数。这种特定类型变量叫做接收者（Receiver）。接收者的概念就类似于其他语言中的this或者 self。

func (接收者变量 接收者类型) 方法名(参数列表) (返回参数) {
    函数体
}
  1. 接收者变量：接收者中的参数变量名在命名时，官方建议使用接收者类型名的第一个小写字母，而不是self、this之类的命名。例如，Person类型的接收者变量应该命名为 p，Connector类型的接收者变量应该命名为c等。
  2. 接收者类型：接收者类型和参数类似，可以是指针类型和非指针类型。如果是指针类型, 对变量的修改会影响外部调用方, 否则只是对接受者副本的修改, 不影响调用方的内部数据.
  3. 方法名、参数列表、返回参数：具体格式与函数定义相同。

什么时候应该使用指针类型接收者

  • 需要修改接收者中的值
  • 接收者是拷贝代价比较大的大对象
  • 保证一致性，如果有某个方法使用了指针接收者，那么其他的方法也应该使用指针接收者。

函数

go里面的函数有以下特点

  • 无需声明原型。
  • 支持不定 变参。
  • 支持多返回值。
  • 命名返回参数。
  • 支持匿名函数和闭包。
  • 函数也是一种类型，一个函数可以赋值给变量。
  • 不支持 嵌套 (nested) 一个包不能有两个名字一样的函数。
  • 不支持 重载 (overload)
  • 不支持 默认参数 (default parameter)。

go的标准函数定义示例, 展现了go函数的部分特点.

func Sum(num ...int) (sum, count int) {
	for _, n := range num {
		sum += n
	}
	count = len(num)
	return sum, count

闭包

闭包是由函数及其相关引用环境组合而成的实体(即：闭包=函数+引用环境)。

“官方”的解释是：所谓“闭包”，指的是一个拥有许多变量和绑定了这些变量的环境的表达式（通常是一个函数），因而这些变量也是该表达式的一部分。

个人理解就是, 一般闭包是在嵌套函数中出现的, 这个外层函数定义了一些变量, 然后函数返回值是内层函数的引用(函数名), 并且内层函数还引用了外层函数的变量. 这个内层函数和外层函数的变量统一称为一个闭包. 

func SumWithInitial(initial int) SumFunc {
	return func (num ...int) int {
		sum := initial
		for _, n := range num {
			sum += n
		}
		return sum
	}
}

func main() {
    f := SumWithInitial(10)
	fmt.Println(f(1, 2, 3, 4, 5), f(2, 3, 4, 5))
}

defer

defer特性

  1. 关键字 defer 用于注册延迟调用。
  2. 这些调用直到 return 前才被执。因此，可以用来做资源清理。
  3. 多个defer语句，按先进后出的方式执行。
  4. defer语句中的变量，在defer声明时就决定了。

defer用途

  1. 关闭文件句柄
  2. 锁资源释放
  3. 数据库连接释放

一个容易出错的使用例子

func tDefer() {
	var whatever [5]struct{}
	for i := range whatever {
		defer func() { fmt.Println(i) }()
	}
}

输出都为4. defer是延迟调用的, 先前只会保存现场, 最后调用的时候i变量为4

稍微修改一下就能符合预期.

for i := range whatever {
    defer func(k int) { fmt.Println(k) }(i)
}
